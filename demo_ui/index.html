    Hybrid Control Stack
    
          
    Classical flight control below, RL mission decisions above
    
          
    
            DroneZ does not ask an LLM to control propellers. PID, sensor fusion, Kalman-style state
            estimation, GPS navigation, and safety rules represent the low-level drone layer. RL/AI
            controls mission-level decisions: assignment, routing, charging, recovery, and priority.

    RL Loop
    
          
    Observation → Action → Step → Reward → Learn
    
          
    
            The OpenEnv loop is still the core: observe drone/order/weather/battery state, choose one JSON
            action, step the environment, receive reward, and improve the policy over many rollouts.

    Honest model-training boundary
    
          
    
            The deterministic improved policy is proven against baselines. A real local GRPO-style run was
            attempted, but it exposed an action-format bottleneck: the model generated invalid actions and
            reward stayed flat. The next training path is action-format SFT plus candidate-choice GRPO.

    Enterprise Deployment Story
    
          
    Simulation first, real-world later
    
          
    
            DroneZ is not low-level motor control. It trains and evaluates mission-level fleet decisions.
            A real deployment would integrate certified flight controllers, PID loops, GPS/IMU fusion,
            aviation safety rules, and sim-to-real validation before touching physical aircraft.
